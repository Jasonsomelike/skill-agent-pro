# 计算机网络教学配图

## 何时生图

- 用户明确要求“画图、生成图片、示意图、拓扑图、配图”；
- TCP 握手、封装解封装、路由转发、ARP/DNS 解析等动态过程用图片能明显降低理解难度；
- 生成 Word 文档时，适合加入一张概念总览图或流程图。

已有知识库整页截图能直接回答时，先展示来源原页；不要用 AI 图替代来源证据。

## 生图流程

1. 先确定图片的单一教学目标，不在一张图中塞入过多概念。
2. 调用 `text2image`，提示词需包含：
   - 白色或浅色背景；
   - 清晰的模块、箭头和方向；
   - 统一配色和留白；
   - 只使用少量大字号中文或数字标签；
   - 不使用装饰性小字、伪代码、品牌 Logo 或水印；
   - 明确这是教学示意图而非真实截图。
3. 输出图片后标注“AI 生成教学示意图”，并用 2～4 句话解释读图顺序。
4. 若工具返回图片文件并已保存到运行时工作区，生成 Word 时把该相对路径传给 `runtime_generate_besti_docx.images`。

## 边界

- 不生成虚假的 Wireshark 截图、设备告警、实验结果或教材页面。
- IP、端口、序号、掩码等精确数字必须先计算或从题目提取，再写入提示词。
- 文生图模型不擅长密集文字；复杂流程优先使用 Mermaid，配图只承担直观解释。
