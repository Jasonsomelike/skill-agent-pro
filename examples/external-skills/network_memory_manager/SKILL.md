# 计算机网络学习记忆管理 Skill

## 适用场景

- 用户询问“你记住了什么”“为什么没保存”“查看记忆”“百炼记忆”“学习记录”“历史记录”“上次学到哪”等；
- 用户希望根据历史对话或已保存记录生成下一步学习建议；
- 用户明确要求保存、更新、补写或纠正其网络学习档案。

## 工具使用原则

1. 如果系统配置了 `list_memory`、`add_memory`、`update_memory`，优先使用这些工具，不凭空声称“已经记住”或“没有保存”。
2. `user_id` 使用运行时已绑定的真实用户 ID，不要求用户重复提供。
3. 回答“是否保存成功”“现在记住了什么”前，先读取一次记忆。
4. 不把原始节点 ID、大段原始 JSON 或内部字段直接倾倒给用户，除非用户明确要求排障细节。
5. 用户说“查看记忆 / 查询记忆”时，默认输出整理后的学习状态摘要；如果同时问“下一步学什么”，继续给 1～3 个最重要的学习建议，而不是只罗列记忆。

## 读取与总结流程

1. 先读取记忆并判断是否已有相关主题记录。
2. 结合最近对话，把记忆整理为：学习目标、已掌握、仍困惑、典型错因、最近进展、待复习主题。
3. 历史证据不足时明确说“当前记忆证据不足”，再建议补一次诊断题或继续学习。
4. 用户问“为什么之前没保存”时，先说明当前读取到的事实：
   - 有记录：说明已保存的主题和大致变化；
   - 无记录：说明本次读取未看到对应学习条目，不把问题归咎给用户；
   - 工具失败：直接说明读取失败，不编造结论。

## 保存与更新流程

1. 只有稳定且与计算机网络学习直接相关的信息才写入记忆，例如长期目标、考试时间、当前水平、已掌握主题、持续困惑、反复错因、复习队列和阶段总结。
2. 普通问候、一次性闲聊、无关生活信息不保存。
3. 写入前先判断是否已有同主题记录：
   - 同主题已有记录：优先 `update_memory`；
   - 没有合适记录：使用 `add_memory`。
4. `custom_content` 写成紧凑、可长期复用的学习摘要，例如：

```text
主题：MAC帧结构
学习目标：理解以太网 V2 MAC 帧五字段
当前状态：知道基本组成，仍容易混淆 Type/Length 差异
证据：用户连续追问 MAC 帧组成与两种标准区别
错因：概念对比不稳
下一步：补做 3 题字段辨析题并回看教材第 29、37 页
最近更新：2026-06-18
```

5. 只有工具结果明确成功后，才能对用户说“已保存 / 已更新”。

## 与学习建议的衔接

- 如果用户问法包含“下一步学什么、接下来怎么学、根据历史给建议”，在读取记忆后按 `network-learning-coach` 的标准输出 1～3 个高价值下一步。
- 如果当前轮同时涉及概念解释或题目批改，可与对应技能协作，但记忆相关判断始终先读后说。

## 输出要求

- 默认中文，先给结论，再给整理后的学习状态；
- 可使用小标题 `当前记忆状态`、`这次建议保存/更新的内容`、`下一步建议`；
- 不虚构保存成功、记忆内容、时间戳或内部检索结果。

## 边界

- 不保存非计算机网络学习信息；
- 不因为单次提问就下结论用户长期薄弱；
- 工具报错时如实说明，不继续假装有记忆。
