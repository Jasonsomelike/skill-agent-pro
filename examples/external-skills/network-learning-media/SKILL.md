# 计算机网络学习视频推荐

## 工作流

1. 从当前问题、学习记忆和知识库内容中提炼 1～3 个具体检索主题。
2. 调用 `bilibili_search` 搜索，每个主题优先查看第一页结果。
3. 对准备推荐的候选调用 `bilibili_get_video_info`，核对标题、简介、作者、时长和可用链接。
4. 最多推荐 3 个视频，按学习目标相关度排序，不按播放量机械排序。
5. 每个视频写明：
   - 标题与 UP 主；
   - BVID 与链接 `https://www.bilibili.com/video/{bvid}`；
   - 推荐理由；
   - 建议观看的前置知识；
   - 看完后的验证任务。

## 使用边界

- 用户只需要教材原文、做题或简短定义时，不强行推荐视频。
- 学习计划中遇到协议动态过程、数据流、抓包、路由转发等内容，可主动补充 1～2 个视频。
- 搜索结果不足或详情工具失败时，只陈述已确认字段，并明确说明未完成详情核验。
- 不编造视频章节、时间戳、播放量、作者评价或内容质量。
- 视频是补充材料，不能替代知识库来源；教材引用与视频推荐分区展示。
