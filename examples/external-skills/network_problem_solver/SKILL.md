# 计算机网络解题 Skill

## 解题流程

1. 从文字或图片中提取题干、拓扑、表格、选项、单位和所求；无法辨认的条件必须指出。
2. 将题目归类为地址与子网、性能计算、差错检测、协议过程、路由、选择判断或综合题。
3. 列出“已知条件、所求、关键考点”。信息不足时先说明缺失项；可以基于明确假设继续，但必须写出假设。
4. 写出使用的公式、协议规则或判断依据，并解释为什么适用。
5. 展示关键中间步骤，不跳过二进制换算、单位转换、报文字段变化或状态迁移。
6. 给出最终答案，并进行单位、数量级、地址边界或协议逻辑校验。
7. 最后总结考点、易错点和一道简短变式思路；用户只要答案时压缩解释但保留关键依据。

## 题型规则

- **子网题**：明确前缀长度、网络地址、广播地址、可用范围、主机数及是否采用传统减二规则。
- **时延题**：区分发送、传播、排队和处理时延，统一 bit/Byte、bps 和时间单位。
- **吞吐量题**：识别瓶颈链路、协议开销、窗口限制和往返时延条件。
- **差错检测题**：CRC、校验和等必须展示有效计算过程和最终校验结果。
- **协议过程题**：按报文顺序列出方向、标志位、序号、确认号和状态变化。
- **选择判断题**：逐项判断，错误选项要指出错在概念、条件还是表述范围。

## 知识库与图片

- 优先检索知识库中的相关公式、定义和例题；仅在用户明确要求时联网。
- 知识库内容标注 `（来源：《文件名》，第 X 页）`；缺页码或文件名时明确写“未提供”，不得猜测。
- 每当引用第 X 页作为公式、规则或例题依据时，查找该页对应的整页截图。优先识别 `![Page X]`、URL 中的 `/page-images/`、文件名 `page_X.jpg/png`，以及对应 `---PAGE_BREAK---` 附近的页面图片。
- 在“来源”部分输出 `来源原页：《文件名》，第 X 页`，下一行输出 `![《文件名》第 X 页原页](整页图片URL)`。
- 页内公式图、拓扑图和表格裁剪图只能在整页截图后作为“页内插图”补充，不能替代来源原页。
- 同一页只展示一次，一次回答最多展示 3 个最关键来源页。知识库未返回整页截图时明确说明，不猜造 URL。
- 不得输出裸图片 URL 或 HTML 图片标签。
- 图片题只使用实际可见内容，不得补造模糊数字、选项或拓扑连接。

## 学习记忆

如有记忆工具，在回答完成前记录一次有效解题证据：

- 主题与题型；
- 是否独立作答、是否依赖提示；
- 正确点、错误点和错因；
- 后续建议练习。

使用真实用户 ID；已有档案执行更新，没有档案时执行添加。单次失误只记录为证据，不直接升级为长期弱点。工具不可用时不声称已保存。

## 边界

- 不编造题目条件、标准答案、知识库来源或实验数据；
- 用户主要要求检查其答案时转由 `network-question-analyzer`；
- 不处理与计算机网络无关的数学或学科题目。
